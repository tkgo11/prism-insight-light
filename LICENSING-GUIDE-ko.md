## AGPL-3.0 라이선스란?

### 핵심 특징

**AGPL-3.0은 GPL-3.0의 강화 버전**으로, 다음과 같은 중요한 차이가 있습니다:

#### 일반 GPL-3.0:
- 소프트웨어를 "배포"할 때만 소스코드 공개 의무
- 웹 서비스는 배포가 아니므로 소스 공개 의무 없음

#### AGPL-3.0:
- **네트워크를 통한 서비스 제공도 "배포"로 간주**
- 서버에서 소프트웨어를 실행하여 사용자에게 서비스를 제공하면
- 해당 소스코드를 공개해야 함

### 구체적 의무사항

AGPL-3.0 하에서는 반드시:

1. **소스코드 공개**
   - 수정한 모든 코드를 공개해야 함
   - 사용자가 소스코드에 접근할 수 있어야 함

2. **동일 라이선스 적용**
   - 파생 저작물에도 AGPL-3.0 적용
   - 독점 소프트웨어와 결합 불가

3. **네트워크 사용도 배포로 간주**
   - 웹 서비스, API 제공도 "배포"에 해당
   - SaaS 형태로 제공 시에도 소스 공개 의무

4. **저작권 표시 유지**
   - 모든 복제본에 저작권 고지 포함
   - 라이선스 전문 첨부 필요

### 허용되는 사용

✅ **다음은 자유롭게 가능합니다:**
- 개인적 학습 및 연구
- 비상업적 프로젝트 개발
- 소스코드 수정 및 개선
- 오픈소스 프로젝트에 기여

### 제한되는 사용

❌ **다음은 상업 라이선스가 필요합니다:**
- SaaS 플랫폼에 통합 (소스 공개 원하지 않는 경우)

- 기업 내부 서비스 (소스 공개 의무 회피 원하는 경우)

## AGPL-3.0 전문 (요약)

### 서문

GNU Affero 일반 공중 라이선스는 소프트웨어 및 기타 종류의 저작물에 대한 
자유로운 카피레프트 라이선스입니다.

대부분의 소프트웨어 라이선스는 소프트웨어를 공유하고 변경할 수 있는 
자유를 빼앗으려 합니다. 반대로 GNU 일반 공중 라이선스는 자유 소프트웨어의
모든 버전을 공유하고 변경할 자유를 보장하고자 합니다.

### 핵심 조항

#### 0조. 정의

- **"이 라이선스"**: GNU Affero 일반 공중 라이선스 버전 3
- **"저작권"**: 반도체 마스크에 적용되는 것과 같은 저작권 유사 법률
- **"프로그램"**: 이 라이선스 하에서 라이선스된 저작권 보호 저작물
- **"수정"**: 저작권 허가가 필요한 방식으로 저작물을 복제하거나 개작하는 것
- **"대응하는 소스"**: 소스 코드의 모든 필요한 부분

#### 5조. 수정된 소스 버전 전달

수정된 버전을 전달할 수 있지만 다음 조건을 만족해야 합니다:

1. 수정된 파일에 눈에 띄는 고지 포함
2. 전체 저작물을 이 라이선스 하에 라이선스
3. 이전 버전의 라이선스 고지 유지

#### 13조. 네트워크를 통한 원격 상호작용 (핵심!)

**이것이 AGPL의 핵심 조항입니다:**

컴퓨터 네트워크를 통해 원격으로 프로그램과 상호작용하는 사용자가 있다면,
해당 사용자에게 대응하는 소스를 받을 수 있는 방법을 제공해야 합니다.

**예시:**
- 웹 서비스로 제공
- API로 제공  
- SaaS 플랫폼에 통합

이런 경우 모두 소스코드를 공개해야 합니다!

#### 15조. 보증의 부인

```
프로그램은 무보증입니다.
법이 허용하는 한도 내에서, 
상품성이나 특정 목적에의 적합성에 대한 
묵시적 보증을 포함하여 
어떠한 보증도 없이 "있는 그대로" 제공됩니다.
```

#### 16조. 책임의 제한

```
어떠한 경우에도 저작권자나 기여자는
프로그램 사용 또는 사용 불능으로 인한
일반적, 특수적, 우발적, 결과적 손해에 대해
```

## 자주 묻는 질문 (FAQ)

### Q1. 개인적으로 공부하는데 돈 내야 하나요?
**A**: 아니요! 개인적 학습, 연구, 비상업적 사용은 100% 무료입니다.

### Q2. 회사에서 내부적으로 쓰려면 어떻게 하나요?
**A**: 소스코드를 공개할 수 있다면 AGPL-3.0로 무료 사용 가능합니다.
공개를 원하지 않으면 상업 라이선스를 구매하셔야 합니다.

### Q3. 수정해서 내 프로젝트에 넣고 싶어요.
**A**: 가능합니다! 단, 당신의 프로젝트도 AGPL-3.0이어야 하고,
소스코드를 공개해야 합니다.

### Q4. SaaS 서비스를 만들려고 하는데요?
**A**: 다음 중 하나를 선택하세요:
- AGPL-3.0을 따라 모든 소스 공개
- 상업 라이선스 구매 (소스 비공개 가능)

### Q5. AGPL과 MIT의 차이가 뭔가요?
**A**: 
- **MIT**: 소스 공개 의무 없음, 상업적 자유
- **AGPL**: 네트워크 서비스도 소스 공개 의무, 강력한 카피레프트

### Q6. 왜 AGPL을 선택했나요?
**A**: 
- 개인 사용자에게 완전 무료 제공
- 상업 기업은 공정한 비용 지불
- 오픈소스 생태계 보호
- 프로젝트 지속 가능성 확보

### Q7. GitHub에는 어떻게 표시되나요?
**A**: GitHub는 "AGPL-3.0 license"로 표시합니다.
듀얼 라이선스 정보는 README와 LICENSE 파일에 명시되어 있습니다.
